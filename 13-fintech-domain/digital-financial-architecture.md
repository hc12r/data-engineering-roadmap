# Digital Financial Architecture

- Layers: channels, APIs, orchestration, services, data, risk/compliance, observability.
- Core systems: ledger, KMS/HSM, identity, payments gateway, messaging/event bus.
- Qualities: resilience, scalability, security, compliance, auditability.
- Patterns: event-driven, CQRS, idempotency, outbox/inbox, saga/transactions.
- Data mesh vs centralized platform trade-offs.
